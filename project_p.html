3100 - Project Description

Main Page

  • This will be our home page for users who aren't signed in.
  • This page will be designed using a side by side border, with the left side having some information on the application and the right side having two buttons: sign up and log in.
  • When the sign up button is clicked a pop up comes up (see more information below).
  • When the log in button is clicked a pop up comes up (see more information below).

Log In

  • This pop up page follows the same template as the sign up pop up.
  • It asks for a username and password, you click the log in button.
  • If the user enters the correct details, they get redirected to the homepage (see more information below).
  • Else an error comes up saying Incorrect password.

Homepage

  • This is the homepage for non-admin users who are signed in.
  • The design of this page is to have three borders; left, center and justify.
  • The left border has information on the current authenticated user:
    • An edit button that creates a pop up (see more information below)
    • An avatar for the user will be displayed.
    • The user's first name and username.
    • Number of followers and following.
    • The rest of the border will be posts made by the user sorted by time.
  • The center border will includes posts made by all users of the application.
    • Right at the top of the page is a form that asks teh user to compose a post.
  • The right border has a log out button whih redirects the user to the main page, this border also has a section cut out for notifications.

Edit User

  • This pop up gives the user the ability to change:
    • The avatar (we will give a selection of avatars to chose from).
    • First and last name, the username can't be changed.

Sign up page

  • The user would be prompted to fill out a form to set up his account.
  • The form will be displayed as a pop-up page with the Home Page in the background.
  • The form will contain the following information to be recieved from the user
    • First Name
    • Last Name
    • Username:
      This will be the name that will appear in the homepage of each user and the identifier for the specific user, it is also a unique name
    • Date Of birth
      To make sure the individual users signing up to the web application are within a certain age group
    • Email:
      This is to verify the user through email
    • Password:
      This is to authenticate the user when he/she is signing in to the web app.
    • Re-enter Password:
      This addition step to ensure that the user wrote the password correctly the first time
  • The form is then submitted and he/she is verified an entry is created in the database corresponding to the inputs within the form.
  • Once the user is signed up for the web application, he/she will be redirected to the login page, and will recive a pop-up message to wait an hour before actually signing in.

Homepage-Logged in (Admin)

    The following will be rendered if the user that is signed in is the adminstrator

  • The page will have the homepages(logged in) layout
  • The admin user has special privileges, where he/she can do the following:
    • Verify users before they are given access to the web app and the database is updated accordingly.
    • Manage the posts being posted by users. This can be done in two ways:
      1. Checking post that have been flagged for specific content
      2. The admin is able to through all the posts.
    • If any changes are made to the posts with the homepage, database is also updated respectively.
