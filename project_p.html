Our project will be an interactive kids game.

The Gaming Platform will be used to test your knowledge/IQ and to see where you rank among others

Main Page

  • This will be our landing page for users who aren't signed in.
  • This page will be designed using flexbox, the left flexbox showcases our logo and the right shows a form for the user to login.
  • Below the login form is an link to sign in if you don't have an existing account.
  • A popup with for sign up comes up when this link is clicked.

Log In

  • The user needs to be previosuly registered to log in.
  • If the user account does not exist or if the user entered wrong information or if the user did not fill a required field, an error box shows up telling the user to try again.
  • If the user was sucessful, they get redirected to the landing page for users who are signed in.

Homepage

  • This is the homepage for admin/non-admin users who have been authorized
  • An automatic popup welcomming the user to the application always shows up, to stop this automatic feature, the user can select the do not show again checkbox and this gets updated in the database through the backend.
  • A leaderboard showing the top 5 users of the application is seen on the left.
  • The leaderboard which is sorted by number of games won includes their rank, their username, number of games won and a smiley button, clicking on this button sends a notiffication to the user saying you have smiled at them.
  • Users can send any of the users on the leaderboard a smile; there is a limit of 5 smiles per user, you can only send 1 smile to a user at a time.
  • If the current authorized user is on the leaderboard, the smiley button will not be seen on their row since you can't send yourself a smile.
  • Each user can only send one smile to a user, if you try to send a smile to a user who has recieved a smile from you a pop up saying the user has already received a smile shows up, else a popup sayign your notifucation has been sent shows up.
  • The notifications aren't real time, hence the home page has to be refreshed to see new notifications.

Change Password

  • When this button is clicked, a popup comes up requiring the user to enter their old password and the new password they would like to change it to.
  • If the old password does not match the user's current password, an error pop up message shows up.
  • If the user did not fill teh reuqired fields properly an error pop up message shows up.
  • If the user was sucessful in changing their password, they get a sucessful popup message.

Sign up

  • The user would be prompted to fill out a form to set up their account.
  • The form will be displayed as a pop-up page with the Home Page in the background.
  • The form will contain the following information to be recieved from the user
    • First Name
    • Username:
      This will be the identifier for the specific user, it is also unique.
    • Password:
      This is to authenticate the user when they are signing in to the web app.
    • Re-enter Password:
      This addition step to ensure that the user wrote the password correctly the first time.
  • If signing up wasn't sucessful, an error pop up appears; this could be caused by:
    • The username already exists in the databse
    • The user did not fill out the required fields properly.
  • If signing up was sucessful, a popup message prompting the user to sign in appears.

Notifications

  • Once a user hovers over the notifications icon, this shows the user all the smiles they have receieved (limit to 5)

Game Categories

  • The user can select different categories and their corresponding difficulty levels.
  • For the purpose of the project, we will only have three categories seen in the Class Demo category:
    • Easy - Animals (Mix and Match):
      Very basic game, that requires the user to select an animal and match it with the matching name or vice-versa by also selecting it.
      The score gets updated incrementally, for every correct match, a score of one gets added to the total score.
      If the user selected a wrong option, the two options get deselected. Once the user gets a score of 6, an automatic popup letting the user know their score has been added to the database and asking them if they want to play again or head back to the home page.
    • Medium - Flags (Drag and Drop):
      This requires the user to drag the country name to the corresponding flag.
      The score gets updated incrementally, for every correct match, a score of one gets added to the total score.
      Once the user gets a score of 6, an automatic popup letting the user know their score has been added to the database and asking them if they want to play again or head back to the home page.
    • Hard - Numbers (Slide):

Admin Page

    If currently registered user is an admin, an icon appears beside the line welcomming them to the page.
  • If the user is an admin, an icon shows up on the welcome back line.
  • When this icon/button is clicked it leads them to the admin page.

Logout

  • When this button is clicked, the user's cookie session expires and they are redirected to the main home page.
